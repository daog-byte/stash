# Plain-English Jargon Glossary

This glossary explains key terms from your current workflow in simple language. Each term appears once only.

## API Route
A backend endpoint inside your app that receives a request and performs an action, like saving data.

## Build
A production readiness check that compiles the app and catches problems before deployment.

## Commit
A saved checkpoint in Git with a message describing what changed.

## Dependency
An external package your project relies on, installed through npm.

## Deploy
Publishing your app so it is live on the internet.

## Environment Variable
A setting value stored outside code, usually for URLs, keys, or environment-specific config.

## Frontend
The part users see and interact with in the browser.

## Git
A version control system that tracks code changes over time.

## GitHub
A cloud platform where Git repositories are stored and shared.

## Localhost
Your own computer running the app locally for testing.

## npm
A package manager used to install dependencies and run project scripts.

## Production
The live environment real users access.

## Repository
A project folder tracked by Git, including history of changes.

## Root Directory
The top-level folder of a project where core config files live.

## Schema
The structure of a database table: columns, data types, and rules.

## Supabase
A backend platform that provides a Postgres database and APIs.

## Table
A spreadsheet-like structure in a database where rows of data are stored.

## Terminal
A text-based interface for running commands.

## Type Check
Validation that code types are correct, used by TypeScript to prevent certain bugs.

## TypeScript
JavaScript with added type safety to catch mistakes earlier.

## Vercel
A hosting platform that deploys your app from GitHub and gives a live URL.

## Workflow
A repeatable sequence of steps to build, test, deploy, and verify changes.
